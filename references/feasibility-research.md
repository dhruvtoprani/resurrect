# Feasibility Delta Research

Research only the unresolved blockers and relevant market or platform changes. Do not write a generic market report.

## Required Questions

- Does the missing technical primitive now exist?
- Has a platform absorbed the original product?
- Can AI eliminate a previously complex subsystem?
- Are required APIs currently available?
- Has implementation cost materially changed?
- Does an existing competitor make the product irrelevant?
- Is there a narrower use case that remains distinctive?

## Source Rules

- Use primary sources for API availability, pricing, SDK support, platform deprecations, and framework support.
- Use current retrieval dates for external claims.
- Separate current facts from inference about impact.
- Cite every external claim with URL and `retrieved YYYY-MM-DD`.

## Output Shape

For each original blocker:

```text
Original blocker:
...

What changed:
...

Evidence:
- ...

Impact:
...

Recommendation:
...
```

If no research is needed, say why and keep the Feasibility Delta focused on repository evidence and owner input.
